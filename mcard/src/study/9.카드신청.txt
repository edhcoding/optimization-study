카드 신청 페이지 - 페이지 단위로 분리 방법 2가지
1. 페이지 단위로 분리
- 여러 페이지로 나누고 각 페이지마다 데이터를 받고 모아서 하나의 데이터로 만드는 방법

몇 가지 문제점이 있음
- 예를들어 각각의 데이터를 바꾸는 부분이 각각의 페이지 안에 숨켜져 있음, 
어떤 데이터가 언제 변하는지 알기 위해서는 각 페이지를 돌아다니면서 어떤 시점에 어떤 데이터를 바꿔야하는지를 다 찾아다녀야함
- 디버깅을 하고 싶어도 각 페이지의 데이터가 바뀌는 부분들에 모두 로그를 심어야함 (너무 코드가 분리되어 있어서 추적이 어렵다는 단점을 가지고 있음)

2. 하나의 페이지 (step 분기 - 컴포넌트 단위로 분리)
- 하나의 페이지가 존재하고 데이터는 이 한 페이지에서 관리하게 됨 global store 사용안함
그리고 이 데이터들은 각각의 step에 따라서 분기가 되는데 예를들어 날짜선텍이면 날짜에 관련된 데이터는 다 거기로 감
만약에 데이터가 채워지면 바로 부모한테 보내면되는데 이렇게 하게 되면 이제 날짜의 변경은 해당 step 컴포넌트가 가져가 돼 이 완성된 데이터를 채우는거는 부모 페이지에서 이루어지기 때문에
각 step 별로 언제 어떤 데이터를 바꾸는지를 쉽게 파악가능함

카드 신청 순서
1. 카드 신청 정보 저장

2. 카드 신청 상태 체크 (2초 마다)
- 준비 / 진행중 / 완료 / 거절

3. 카드 신청 완료
- 거절시 카드 상세페이지로 이동
- 완료시 내정보 페이지로 이동

쿼리 스트링 값을 더 쉽게 가져올 수 있는 라이브러리 (qs)
- yarn add qs
- yarn add -D @types/qs
