전역 상태관리를 하는 기준은?
아래 두 가지 중 하나라도 해당되면 전역 상태관리를 사용함 그 외에는 깊게 고민해봐야함
- 여러 페이지를 거쳐서 공통적으로 값을 다뤄야 할 때
ex) 유저 정보 값, 신청 폼 값
- 컴포넌트의 깊이가 너무 깊어졌을때

Context API 와 Recoil
1. Context API
- React 에서 기본 지원
- 사용 간단
- 성능 최적화는 직접 관리해야함

2. Recoil (yarn add recoil)
- 단순하다 => 코드의 복잡도를 낮춰줌
- 비동기 상태 관리 업데이트를 쉽게 처리 가능(selector 사용시 캐싱도 지원)
- 변경된 상태값에 관련된 컴포넌트만 업데이트
